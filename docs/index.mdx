# The computer was always the missing product

Agent frameworks and harness stacks keep growing layers.  
They were **converging on a process virtual machine**. We build that machine explicitly.

## One sentence

> A **World** is a durable computer. A **host** is disposable CPU.  
> **Processes** are the unit of work. **Syscalls** are the only non-determinism.  
> **Fork** is how you explore. **Mesh** is how many worlds become a multi-verse.

## Not another stack

| Stack word | Here |
|---|---|
| Agent framework | **PVM processes** on a World |
| Harness / runner | **Host** (`pvm` or your binary) |
| Memory service | **Same world** (KV + tables) |
| Subagent sandbox | **Forked world** (or child process) |
| Multi-agent platform | **Mesh of worlds** (next) |

We **borrowed** libSQL (medium) and Duroxide (replay). We are **not** promoting those projects as the product.  
They are engines inside a **World kernel**.

## What you can do today

| Benefit | Capability |
|---|---|
| Survive host death | Journal primacy |
| Debug stuck work | Health / ps / why / trace |
| Recover failures | Healing + audit |
| Change logic as data | Definitions + `if` / `select` |
| Speculative work safely | Fork → discard / promote |
| Import only what worked | Selective promote (mesh-friendly) |
| Name many worlds | Mesh peers & refs (foundation) |
